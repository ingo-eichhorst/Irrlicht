Working-state icon — 5 alternatives

v0.4.5 shipped a heartbeat halo for the working state. The animation breaks on macOS (frame degradation) and sometimes on the web, leaving only the tiny inner 2.6 px dot visible — too small to read at a glance. Each alternative below stays clearly visible with animation, without animation, and at the smallest 12 px row size. Compare in the size strip and the in-row context, then toggle reduced-motion to confirm the static fallback is also readable.

A · Rotating arc spinner

Pro: CSS transform: rotate is the most reliable animation primitive everywhere — works identically in WebKit, Blink, SwiftUI .rotationEffect. Substantial stroke width (3 of 20 viewBox) means it stays legible even at 12 px.
Con: Universally recognized as "loading" — may read more like a network spinner than a working agent. Static fallback is a frozen partial arc (still visible, but ambiguous).

Pro: Reads as "active output / processing." Each bar always visible at minimum 35 % height, so a frozen state still reads as 4 standing bars. CSS scaleY + animation-delay only — trivial to mirror in SwiftUI.
Con: At 12 px width, 4 bars means each bar is ~2 px wide — gap is tight. Could drop to 3 bars at the smallest size if needed.

Pro: Reads as "thinking…" — the same vocabulary chat UIs use for "agent typing." All three dots always visible (opacity floor 0.35), so the static fallback is three solid purple dots at full opacity — completely legible.
Con: Three dots is wider than tall — needs a 3:1 aspect at small sizes (the cell still renders at 12×12 here, dots are circular within it). Reads less as a glyph and more as a row of indicators.

Pro: Largest possible footprint (r=7 of 20 viewBox = 70 % of cell). Opacity-only animation never shrinks the shape — even with animation completely broken or disabled, you see a big solid purple disc. Closest match to the v0.4.5 "circle" vocabulary so existing-user recognition is preserved.
Con: The least "in motion" of the five. Easy to miss the breathe at small sizes — reads almost as a static state when ambient.

Pro: On-brand — uses the v0.4.5 gradient-flame mark in miniature. Subtle flicker (opacity + 8 % scale) reads as "burning / active" without violent motion. Static fallback is the brand mark itself — strong identity anchor.
Con: The brand flame also appears in the menu-bar idle icon and the nav-bar — using it again as a per-row state risks visual repetition / loss of distinct meaning. At 12 px the gradient stops compress into a flat mid-tone purple, which mutes the identity payoff.
